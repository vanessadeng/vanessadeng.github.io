Getting Started with AWS

AWS Overview

  • Amazon Web Services (AWS) is a comprehensive cloud platform offering over 30 services.
  • It has revolutionized IT by providing scalable, on-demand server and service access.
  • This course prepares for the Certified Developer exam – DVA-C02, covering from AWS basics to advanced features.

AWS Cloud History

  • 2002: AWS started internally.
  • 2004: Launched publicly with SQS.
  • 2006: Re-launched with SQS, S3, and EC2.
  • As of 2019, AWS had a substantial market share and over 1,000,000 active users.

AWS Global Infrastructure

  • Consists of AWS Regions, Availability Zones (AZs), Data Centers, and Edge Locations.
  • Regions are clusters of data centers (e.g., us-east-1).
  • Region selection is influenced by data governance, latency, service availability, and pricing.

AWS Identity & Access Management (IAM)

  • IAM is a global service for secure AWS management.
  • Root accounts are created by default and should not be shared.
  • IAM Users can be organized into Groups with permissions assigned via JSON policies.
  • IAM Roles are used by AWS services to perform actions on behalf of users.
  • Security tools and best practices include enforcing MFA and using IAM Credentials Report & IAM Access Advisor for auditing.

Amazon EC2 – Basics

  • EC2 offers scalable virtual machines.
  • Supports different instance types like General Purpose (t2.micro), Compute Optimized, Memory Optimized, and Storage Optimized.
  • EBS Volumes offer persistent network drive storage.
  • Different purchasing options are available: On-Demand, Reserved, Savings Plans, Spot Instances, Dedicated Hosts, and Capacity Reservations.

Amazon EC2 – Instance Storage

  • EBS Volumes are network drives that offer persistent data storage.
  • Various types include General Purpose (SSD), Provisioned IOPS (SSD), and Hard Disk Drives (HDD).
  • EBS Snapshots back up EBS volumes for restoration or volume creation.
  • AMIs provide pre-configured templates for EC2 instances.
  • EC2 Instance Store offers temporary, high-performance storage.

High Availability & Scalability

  • Scalability can be vertical (size increase) or horizontal (adding instances).
  • High Availability is achieved by running applications across multiple data centers or AZs.
  • Load balancers like Classic, Application, Network, and Gateway Load Balancer distribute traffic across EC2 instances to improve fault tolerance and scalability.

Amazon EC2 Basics

  • EC2 instances offer scalable computing capacity in AWS.
  • They come in various types for different applications:
    • General-purpose
    • Compute-optimized
    • Memory-optimized
    • Storage-optimized
  • EBS Volumes offer persistent block storage volumes for EC2 instances.
  • EBS Snapshots allow backing up volumes in S3 for durability.

High Availability & Scalability

  • Scalability can be vertical (scale up/down) or horizontal (scale out/in).
  • High Availability involves running instances in multiple AZs.

Elastic Load Balancers (ELB)

  • Distribute traffic across multiple instances and AZs.
  • Perform health checks on instances.
  • Support for HTTP/HTTPS protocols.

Security Groups and IAM

  • Act as a virtual firewall for instances to control inbound and outbound traffic.
  • IAM roles are used to securely give permissions to AWS services and resources.

EBS and AMIs

  • EBS offers SSD and HDD volumes for different use cases:
    • High performance
    • Cold data storage
  • AMIs allow the creation of pre-configured instances.

AWS Auto Scaling

  • Ensures the correct number of EC2 instances are running to handle the load of your application.
  • Uses Launch Configurations and Launch Templates to determine instance settings.

Load Balancers Types

  • Classic Load Balancer (v1): for simple load balancing of HTTP/HTTPS traffic.
  • Application Load Balancer (v2): for HTTP/HTTPS traffic, with advanced request routing.
  • Network Load Balancer (v2): for ultra-high performance and static IP addresses for TCP traffic.
  • Gateway Load Balancer: for deploying, scaling, and managing third-party virtual appliances.

Databases and Cache

  • RDS: for relational databases managed by AWS with support for multiple engines.
  • DynamoDB: for managed NoSQL databases with seamless scaling and single-digit millisecond latency.
  • ElastiCache: to deploy, operate, and scale an in-memory cache in the cloud, supporting both Redis and Memcached.

Domain and DNS Services

  • Route 53: for a highly available and scalable DNS web service.

Monitoring and Auditing

  • CloudWatch: monitors AWS resources and applications, collects and tracks metrics, collects and monitors log files, and sets alarms.
  • CloudTrail: provides a record of actions taken by a user, role, or an AWS service in your account.

Deployment and Management

  • AWS Elastic Beanstalk: for deploying and scaling web applications and services.
  • AWS CloudFormation: for creating and managing a collection of related AWS resources.

Developer Tools

  • AWS CodeCommit, CodeBuild, CodeDeploy, and CodePipeline: for continuous integration, continuous delivery, and quick development & deployment on AWS.

Microservices and Serverless

  • AWS Lambda: lets you run code without provisioning or managing servers.
  • Amazon API Gateway: for creating, publishing, maintaining, monitoring, and securing REST, HTTP, and WebSocket APIs.

Decoupling Applications

  • Amazon SQS and SNS: for decoupling and scaling microservices, distributed systems, and serverless applications.

AWS Shared Responsibility Model

  • AWS is responsible for the security of the cloud, while customers are responsible for security in the cloud.

EBS (Elastic Block Store) and EFS (Elastic File System):

  • EBS provides persistent block storage for EC2 instances.
  • Volumes are specific to availability zones and come in types like General Purpose (gp2/gp3), Provisioned IOPS (io1/io2), and HDD (st1/sc1).
  • EBS Snapshots allow for backups and restorations of volumes.
  • AMIs offer pre-configured templates for EC2 instances.
  • EC2 Instance Store grants temporary, high-performance storage.
  • EFS is a managed file system for multiple EC2 instances, ideal for shared file systems.

High Availability & Scalability:

  • Vertical scalability: increasing instance size.
  • Horizontal scalability (elasticity): adding more instances.
  • High availability: distributing instances across multiple availability zones.
  • Load Balancers: distribute traffic, enhance fault tolerance, and improve availability.
  • Types of ELBs: Classic, Application, Network, and Gateway Load Balancers.

Elastic Load Balancing (ELB):

  • Distributes incoming application traffic across multiple targets.
  • Monitors health of targets, routing only to the healthy ones.
  • Provides SSL/TLS decryption and user authentication.

AWS Identity & Access Management (IAM):

  • Manages secure access to AWS services and resources.
  • Defines Users and Groups for access management.
  • Uses JSON policies for permissions, adhering to the least privilege principle.
  • IAM Roles delegate permissions without sharing credentials.
  • Management tools include IAM Credentials Report and Access Advisor.

EC2 Instances:

  • Purchase options: On-Demand, Reserved, Savings Plans, Spot Instances, and Dedicated Hosts.
  • Instance types: General Purpose (t2.micro), Compute Optimized, Memory Optimized, and Storage Optimized.
  • Denoted by instance family (e.g., M5), generation, and size (e.g., 2xlarge).
  • Security Groups act as virtual firewalls for instances.

EC2 Purchasing Options:

  • On-Demand Instances: Pay for what you use without long-term commitment.
  • Reserved Instances and Savings Plans: Discounted rates for a commitment period.
  • Spot Instances: Bid on unused capacity at lower prices but with potential for termination.
  • Dedicated Hosts: Physical servers dedicated for compliance and licensing needs.
  • Capacity Reservations: Ensure dedicated capacity in a specific Availability Zone.
